Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build at your own.

Recursion is a programming concept where a function calls itself. This can reduce a complex problem into simpler sub-problems, until they become straightforward to solve. In this mini project, you’ll build a decimal-to-binary converter using JavaScript. You’ll practice the fundamental concepts of recursion, explore the call stack, and build out a visual representation of the recursion process through an animation.

User Stories:

  • You should have a constant variable numberInput that stores the DOM element with id number-input. This will be used to get the decimal number entered by the user.
  • You should have a constant variable convertBtn that stores the DOM element with id convert-btn. This button triggers the conversion process.
  • You should have a constant variable result that stores the DOM element with id result. This will display the binary output or animation messages.
  • You should have a constant variable animationContainer that stores the DOM element with id animation-container. This will be used to display call stack animation frames for a special case.
  • You should have an array constant animationData containing objects with keys inputVal, addElDelay, msg, showMsgDelay, and removeElDelay. This is used to control the step-by-step animation for input 5.
  • You should have a function decimalToBinary that:
    • Accepts an integer input.
    • If the input is 0 or 1, returns it as a string (base case).
    • Otherwise, recursively calls itself with Math.floor(input / 2) and appends (input % 2) to the returned value.
  • You should have a function showAnimation that:
    • Sets the result element text to "Call Stack Animation".
    • Iterates over animationData to:
      • Add a paragraph element showing decimalToBinary(inputVal) after addElDelay ms.
      • Replace the paragraph text with the msg after showMsgDelay ms.
      • Remove the paragraph after removeElDelay ms.
    • After 20000 ms, updates result to show the binary conversion of 5.
  • You should have a function checkUserInput that:
    • Parses the value of numberInput as an integer.
    • If the input is missing, not a number, or negative, shows an alert with the message "Please provide a decimal number greater than or equal to 0".
    • If the input is exactly 5, calls showAnimation and returns.
    • Otherwise, calls decimalToBinary and displays the result in result, then clears the input field.
  • You should add an event listener on convertBtn to call checkUserInput when clicked.
  • You should add an event listener on numberInput for the "keydown" event so that pressing the Enter key also triggers checkUserInput.
  • You should have commented-out alternative implementations:
    • Array-based iterative approach tracking inputs, quotients, and remainders.
    • Recursive countdown demonstration function countDownAndUp.
    • Optimized iterative approach using string building.

Tests:

  • 1. You should have a variable numberInput assigned to the element with id number-input.
  • 2. You should have a variable convertBtn assigned to the element with id convert-btn.
  • 3. You should have a variable result assigned to the element with id result.
  • 4. You should have a variable animationContainer assigned to the element with id animation-container.
  • 5. You should have a constant animationData array with the required objects and keys.
  • 6. You should have a function decimalToBinary that implements the recursive conversion correctly.
  • 7. decimalToBinary(0) should return "0".
  • 8. decimalToBinary(1) should return "1".
  • 9. decimalToBinary(5) should return "101".
  • 10. showAnimation() should display call stack frames for input 5 and end with "101" after 20000 ms.
  • 11. checkUserInput() should alert if the input is invalid.
  • 12. If the input is 5, checkUserInput() should trigger the animation.
  • 13. For other valid inputs, checkUserInput() should display the binary conversion result.
  • 14. Clicking convertBtn should trigger checkUserInput().
  • 15. Pressing Enter while focused on numberInput should trigger checkUserInput().
