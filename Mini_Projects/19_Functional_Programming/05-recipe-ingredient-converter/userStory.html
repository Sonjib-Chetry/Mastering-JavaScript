Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build at your own.

In this mini project, you will apply the functional programming concepts you have learned in the previous lectures to build an ingredient converter app.

User Stories:

  • You should define a constant conversionTable that stores the conversion rates between units:
    • Units should include: cup, gram, ounce, and teaspoon.
    • Each unit should have nested key-value pairs for conversion to other units.
  • You should define a curried function convertQuantity that:
    • Takes a fromUnit as the first argument.
    • Returns a function that takes a toUnit as the second argument.
    • Returns another function that takes quantity as the third argument.
    • Looks up the conversion rate from conversionTable and returns the converted quantity.
  • You should define another curried function adjustForServings that:
    • Takes baseQuantity as the first argument.
    • Returns a function that takes newServings as the second argument.
    • Adjusts the quantity proportionally to match the new servings.
  • You should define a function processIngredient that:
    • Takes baseQuantity, baseUnit, newUnit, and newServings as parameters.
    • Adjusts the quantity for the new servings using adjustForServings.
    • Converts the adjusted quantity to the new unit using convertQuantity.
    • Returns the converted value rounded to two decimal places.
  • You should connect the DOM elements:
    • ingredientName – text input for ingredient name.
    • ingredientQuantity – number input for ingredient amount.
    • unitToConvert – dropdown for selecting the base unit.
    • numberOfServings – number input for servings.
    • recipeForm – the form containing the inputs.
    • resultList – the list to display the converted ingredient values.
  • You should define an updateResultsList function that:
    • Clears the resultList before updating it.
    • Loops over all units in the units array except the currently selected unit.
    • For each unit, calls processIngredient and appends the converted result to the list.
  • You should add a submit event listener to recipeForm that:
    • Prevents default form submission.
    • Calls updateResultsList to display conversion results.

Tests:

  • 1. You should define a conversionTable object with correct unit mappings.
  • 2. You should have a curried function convertQuantity that converts between units.
  • 3. convertQuantity("cup")("gram")(2) should return 480.
  • 4. You should have a curried function adjustForServings that adjusts quantities.
  • 5. adjustForServings(4)(6) should return 24.
  • 6. You should have a function processIngredient that adjusts for servings and converts units.
  • 7. You should display conversion results for all other units in updateResultsList.
  • 8. Submitting the form should update the resultList with correct conversions.
