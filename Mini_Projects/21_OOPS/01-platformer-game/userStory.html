Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build at your own.

Coding a game is a great way to grasp fundamental programming principles, while also creating an interactive gaming experience. In this platformer game project, you'll continue to learn about classes, objects, inheritance, and encapsulation. You'll also learn how to design and organize game elements efficiently and gain insights into problem-solving and code reusability.

User Stories

  • Create a Start Screen with a button #start-btn. When clicked, it should hide the start screen and display a <canvas> element where the game runs.
  • Initialize the canvas to the full width and height of the browser window. Get the 2D context using canvas.getContext("2d").
  • Define a Player class that:
    • Has properties: position, velocity, width, height.
    • Has a draw() method to render the player as a rectangle.
    • Has an update() method to apply gravity, update position, and prevent leaving canvas boundaries.
  • Define a Platform class that:
    • Draws a green rectangle on the canvas at given coordinates.
    • Acts as a surface where the player can land.
  • Define a CheckPoint class that:
    • Draws a gold rectangle at given coordinates.
    • Can be “claimed” (hidden) when the player collides with it.
    • Displays a checkpoint message on screen when collected.
  • Create arrays of platform positions and checkpoint positions. Loop through them to create Platform and CheckPoint objects.
  • Write an animate() function that:
    • Clears the canvas every frame.
    • Draws all platforms and checkpoints.
    • Updates the player position and movement.
    • Handles collisions with platforms and checkpoints.
    • Scrolls the world (platforms and checkpoints shift) as the player moves left/right.
  • Add keyboard event listeners for keydown and keyup:
    • ArrowLeft: move player left.
    • ArrowRight: move player right.
    • ArrowUp or Space: make player jump upward.
  • Show a message "You reached a checkpoint!" when hitting a checkpoint, and "You reached the final checkpoint!" when reaching the last checkpoint. Disable movement after the last checkpoint.

Test Cases

  • Start Game: Clicking #start-btn hides the start screen and shows the canvas.
  • Player Rendering: On load, a blue square appears at the starting position.
  • Player Movement: Pressing ArrowRight moves the player right, ArrowLeft moves left.
  • Jumping: Pressing ArrowUp or Space moves the player upward briefly (jump).
  • Gravity: The player falls down if not on a platform, but stops when landing on the ground or a platform.
  • Boundaries: The player cannot move outside the left or right edges of the canvas.
  • Platform Collision: The player lands on platforms and does not fall through them.
  • Checkpoint Collision: When touching a checkpoint, it disappears and a message is displayed.
  • Checkpoint Order: Later checkpoints only activate if previous ones were claimed.
  • Final Checkpoint: When the last checkpoint is collected, message “You reached the final checkpoint!” is shown and movement is disabled.
  • World Scrolling: As the player moves right, platforms and checkpoints shift left, creating scrolling effect. Moving left shifts them back.
