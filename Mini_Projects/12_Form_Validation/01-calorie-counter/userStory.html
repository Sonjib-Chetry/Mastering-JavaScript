Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build at your own.

Sometimes when you're coding a web application, you'll need to be able to accept input from a user. In this calorie counter mini project, you'll practice how to validate user input, perform calculations based on that input, and dynamically update your interface to display the results. You'll also practice basic regular expressions, template literals, the addEventListener() method, and more.

User Stories

  • As a user, I want to input a daily calorie budget.
  • As a user, I want to add multiple food or exercise entries categorized by meal types (Breakfast, Lunch, Dinner, Snacks, Exercise).
  • As a user, I want to see dynamically added input fields for food or exercise entries based on my selection.
  • As a user, I want to calculate total calories consumed and burned, and see whether I’m in surplus or deficit.
  • As a user, I want the app to prevent invalid input like scientific notation or negative numbers.
  • As a user, I want to clear all fields and reset the form with a single click.

Test Cases

  • ✅ The page should contain an input with id="budget" for entering the calorie budget.
  • ✅ There should be a select element with id="entry-dropdown" to choose the meal type.
  • ✅ Clicking the button with id="add-entry" should append new labeled text and number input fields inside the corresponding `.input-container`.
  • ✅ The form submission should calculate and display:
    • Budgeted calories
    • Consumed calories (sum of Breakfast, Lunch, Dinner, Snacks)
    • Burned calories (from Exercise)
    • Remaining calories and whether it’s a Deficit or Surplus
  • ✅ When an invalid calorie input (e.g., 1e6) is entered, an alert with “Invalid Input” should appear and calculation should abort.
  • ✅ Clicking the "Clear" button with id="clear" should:
    • Clear all input fields
    • Hide the result section
  • ✅ When valid values are entered and submitted, the results should appear in the #output element and include calorie stats and deficit/surplus.
  • ✅ The `getCaloriesFromInputs()` function should sanitize values and convert them to numbers before summing.
  • ✅ Each entry should have unique ids based on entry type and index (e.g., breakfast-1-name, lunch-2-calories).
