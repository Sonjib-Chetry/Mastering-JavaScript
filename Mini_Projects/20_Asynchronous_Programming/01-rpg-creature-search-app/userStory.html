User Stories

  • You should have an input element with an id of search-input, and is required.
  • You should have a button element with an id of search-button.
  • You should have an element with an id of creature-name.
  • You should have an element with an id of creature-id.
  • You should have an element with an id of weight.
  • You should have an element with an id of height.
  • You should have an element with an id of types.
  • You should have an element with an id of hp.
  • You should have an element with an id of attack.
  • You should have an element with an id of defense.
  • You should have an element with an id of special-attack.
  • You should have an element with an id of special-defense.
  • You should have an element with an id of speed.
  • When the #search-input element contains the value Red and the #search-button element is clicked, an alert should appear with the text "Creature not found".
  • When the #search-input element contains the value Pyrolynx and the #search-button element is clicked, the values in the UI should be:
    • #creature-name: PYROLYNX
    • #creature-id: #1 or 1
    • #weight: Weight: 42 or 42
    • #height: Height: 32 or 32
    • #hp: 65
    • #attack: 80
    • #defense: 50
    • #special-attack: 90
    • #special-defense: 55
    • #speed: 100
  • When searching for Pyrolynx, a single element should be added within the #types element containing FIRE. The #types content should be cleared between searches.
  • When the #search-input element contains the value 2 and the #search-button element is clicked, the values in the UI should be:
    • #creature-name: AQUOROC
    • #creature-id: #2 or 2
    • #weight: Weight: 220 or 220
    • #height: Height: 53 or 53
    • #hp: 85
    • #attack: 90
    • #defense: 120
    • #special-attack: 60
    • #special-defense: 70
    • #speed: 40
  • When searching for ID 2, two elements should be added within the #types element containing WATER and ROCK. The #types content should be cleared between searches.
  • When the #search-input element contains an invalid creature name, and the #search-button is clicked, an alert should appear with the text "Creature not found".
  • When the #search-input element contains a valid creature ID and the #search-button is clicked, the UI should be filled with the correct data.
