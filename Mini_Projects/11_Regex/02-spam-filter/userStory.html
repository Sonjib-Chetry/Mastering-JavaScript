Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build at your own.

In this mini project, you'll use capture groups, positive lookaheads, negative lookaheads, and other techniques to match any text you want.

User Stories

  • You should have a textarea or input element with the id="message-input" to receive user messages.
  • You should have a p or similar display element with the id="result" to show spam detection output.
  • You should have a button element with the id="check-message-btn" that triggers the spam check when clicked.
  • You should create five regular expressions to detect spam patterns such as:
    • "please help" or "assist me"
    • Currency references like "1000 dollars", "5 million dollars"
    • "free money" with substitutions like "fr33 m0ney"
    • "stock alert" with altered characters like "5t0ck @lert"
    • "dear friend" variations like "d34r fr1end"
  • You should store all the regex patterns inside a denyList array.
  • You should create a function isSpam(msg) that returns true if any of the spam patterns match the message.
  • When the check button is clicked, you should check the value of the input field.
  • If the field is empty, show an alert that says "Please enter a message."
  • If the message contains spam, display the message: "Oh no! This looks like a spam message."
  • If the message is clean, display the message: "This message does not seem to contain any spam."
  • After checking the message, the input field should be cleared.

Test Cases

  1. The textarea or input with id="message-input" should exist in the HTML.
  2. The p or div with id="result" should exist in the HTML.
  3. The button with id="check-message-btn" should exist in the HTML.
  4. Entering "Please help me now!" should return the spam result.
  5. Entering "You have won 5 million dollars!" should return the spam result.
  6. Entering "Claim your fr33 m0ney now!" should return the spam result.
  7. Entering "Hot 5t0ck @lert!" should return the spam result.
  8. Entering "D34r fr1end, I need your help." should return the spam result.
  9. Entering "Let's meet for lunch today." should return the clean result.
  10. If the input field is empty and the button is clicked, an alert should be triggered saying: "Please enter a message."
  11. After clicking the check button, the input field should be cleared regardless of spam or clean result.
