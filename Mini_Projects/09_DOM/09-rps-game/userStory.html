Welcome to my **90 Days of JavaScript Revision Challenge**!

Please read the script file and understand the project and try to build your own.

In this mini project, you will continue to practice working with the DOM and event handling by building a Rock, Paper, Scissors game. The HTML and CSS boilerplate for the game has been provided for you. On the screen, you will see the rules to the game, the player and computer scores and the buttons for the user to select Rock, Paper, or Scissors. Your task over the next few steps is to build the JavaScript functionality that generates a computer choice, compares it to the player's choice, and updates the scores accordingly. Take some time to look through the HTML provided and familiarize yourself with the structure of the game.

User Stories

  • You should define a function getRandomComputerResult() that randomly returns one of "Rock", "Paper", or "Scissors".
  • You should define a function hasPlayerWonTheRound(player, computer) that returns true if the player beats the computer based on standard game rules.
  • You should initialize two variables playerScore and computerScore to 0 to keep track of the scores.
  • You should define a function getRoundResults(userOption) that:
    • Gets a random computer choice using getRandomComputerResult().
    • Determines the outcome using hasPlayerWonTheRound().
    • Updates the score accordingly and returns a string describing the result.
  • You should display the updated round result in the element with ID results-msg.
  • You should update the score elements player-score and computer-score after each round.
  • You should show a winner message inside the element with ID winner-msg when a score of 3 is reached by either player or computer.
  • You should hide the game buttons and display a reset button (ID reset-game-btn) when the game ends.
  • You should define a function resetGame() to reset scores, messages, and visibility of game elements.
  • You should attach event listeners to the game option buttons (rock, paper, scissors) and the reset button to call the appropriate functions.

Test Cases

  1. The getRandomComputerResult() function should return one of "Rock", "Paper", or "Scissors".
  2. The hasPlayerWonTheRound() function should return true for winning combinations like Rock vs Scissors, and false for losing ones.
  3. When the player selects an option, the getRoundResults() function should return a string containing either "Player wins!", "Computer wins!" or "It's a tie!".
  4. The playerScore or computerScore should increment appropriately depending on the round result.
  5. The round result message should appear in the element with ID results-msg.
  6. The score values should update inside player-score and computer-score elements.
  7. When either score reaches 3, the winner-msg element should display the winning message.
  8. When the game ends, the options-container should be hidden and the reset-game-btn should be visible.
  9. Clicking the reset button should reset both scores to 0 and clear all messages.
  10. After reset, the options-container should be visible again and reset-game-btn should be hidden.
